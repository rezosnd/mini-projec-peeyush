### 3. System Architecture
The application follows a **Modular Server-Action Architecture**:
- **Server Components**: Dashboards fetch data directly from the PostgreSQL instance, ensuring zero-latency data hydration.
- **Server Actions**: All mutations (Assigning tasks, updating progress, creating groups) are handled by secured server-side functions.
- **Middleware**: A centralized security layer intercepts every request to verify JWT tokens and enforce role-based access (Teacher vs. Student).

---

### 4. Database Schema
The relational structure ensures data consistency across the portal:

1.  **Users (`users`)**: Stores identities, roles (ADMIN, TEACHER, STUDENT), and hashed credentials.
2.  **Student Groups (`student_groups`)**: Manages collaborative units created by students.
3.  **Group Members (`student_group_members`)**: Maps students to their respective groups.
4.  **Tasks (`tasks`)**: The core entity. Stores title, description, progress (0-100%), and links to PPT/Reports/GitHub.
5.  **To-Dos (`task_todos`)**: Granular milestones within a task that can be toggled by students.

### 5. Features & Functionalities

#### 👨‍🏫 For Teachers (The Instructor)
*   **Managed Dashboard**: A two-column interface for oversight and control.
*   **Selective Assignment**: Capability to assign projects to an **Individual** or a **Group**.
*   **Live Monitoring**: Real-time progress bars showing exactly how much of a project is completed by students.
*   **Asset Review**: Instant access to submitted PPTs, Reports, and Source Code repositories.

#### 👨‍🎓 For Students (The Alpha)
*   **Stream Management**: View all personal and group-assigned projects.
*   **Collaboration Hub**: Create project groups and add members via Roll Numbers.
*   **Progress Tracking**: Update completion percentages and manage a dynamic to-do list for every task.
*   **Secure Submission**: Dedicated portal to upload project deliverables.

#### 🛠️ For Admins (System Control)
*   **Entity Management**: Full oversight of all registered users.
*   **Identity Control**: Ability to rename users, update roll numbers, or reset passwords.
