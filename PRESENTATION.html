KIITease

Managed Portal for Academic Project Excellence

A unified ecosystem designed for KIIT University to streamline the collaboration between student groups and faculty mentors through high-end aesthetics and real-time synchronization.

Next.js 15 Neon DB Full-Stack SSR
Analysis

Literature Review

Generic PM Tools

Trello and Jira lack the specific academic hierarchy needed for student-teacher dynamics and grade-linked task tracking.

Learning Barriers

Manual tracking methods result in "Dead Zones" where faculty cannot monitor real-time contributions across parallel groups.

Identification

Problem Formulation

The core challenge lies in the fragmentation of student collaborative data.

  • ⚡ Visibility Gap: Teachers lack a dashboard for cross-sectional progress.
  • ⚡ Group Chaos: Manual group forming leads to administrative overhead and errors.
  • ⚡ Action Lag: Delay in task assignment and feedback loops reduces student throughput.
Unique Value

Novelty

Managed Portal Logic

Role-specific UI routing that transforms the interface based on user authorization, providing tailored experiences for Students vs. Teachers.

Visual Milestone Map

A glassmorphic progress layer that provides instant visual feedback on project completion metrics across the entire collective.

Architecture

Proposed Method

Leveraging Serverless Relational Architecture for maximum scalability and type safety.

Data Flow

Optimistic UI updates via React Server Actions minimize perceived latency while maintaining consistency in Neon DB.

Group Synergy

Dynamic many-to-many relationship mapping between Enrollments, Groups, and Tasks for high-fidelity tracking.
